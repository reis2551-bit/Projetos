⏰
Barra de Urgência no topo
A frase "Apenas 3 vagas disponíveis esta semana" cria escassez percebida, um dos gatilhos psicológicos mais eficazes em marketing. Visitantes que não se sentem pressionados adiam a decisão — e raramente voltam.
↑ Taxa de contacto imediato
💬
WhatsApp como CTA principal
Estudos de serviços locais mostram que o WhatsApp converte 3–4× mais do que formulários. É familiar, instantâneo e sem fricção. A mensagem pré-preenchida elimina o "não sei o que escrever", reduzindo o abandono.
3–4× conversão vs. formulário
📌
Navbar fixa com telefone visível
O número de telefone e o botão de WhatsApp estão sempre visíveis no topo, independentemente de onde o utilizador estiver na página. Elimina a necessidade de procurar o contacto — um atrito silencioso que perde clientes.
Contacto sempre a 0 cliques
🔁
Múltiplos CTAs ao longo da página
Existem botões de acção após cada secção de serviço, a meio da página e no final. Diferentes utilizadores tomam decisões em momentos diferentes — garantir que há sempre um CTA acessível no momento de decisão é fundamental.

📱
Mobile-first design
O site foi desenhado primeiro para ecrãs de 375px e adaptado para desktop — não o inverso. Grelhas, espaçamentos e tamanhos de texto foram todos calibrados para leitura e toque em telemóvel. O botão de WhatsApp flutuante está sempre visível no canto inferior direito.
⚡
Sem frameworks, sem bibliotecas
O site não usa React, Vue, WordPress, jQuery nem qualquer dependência externa. É HTML, CSS e JavaScript puro. Resultado: carregamento instantâneo, sem risco de actualizações quebradas, sem mensalidades de plugins e fácil de hospedar em qualquer lado.
🎯
Botão WhatsApp flutuante fixo
Em mobile, o botão verde do WhatsApp está fixo no ecrã a toda a altura. É o ponto de contacto de menor fricção possível: um toque, conversa aberta. Em serviços onde a decisão é emocional e urgente, isto é determinante para não perder o momento de intenção.
🔒
Hospedagem GitHub Pages (grátis e fiável)
O site está publicado no GitHub Pages com deploy automático via GitHub Actions — cada alteração ao ficheiro HTML é publicada em segundos, sem FTP, sem painel de controlo. Uptime de 99.9%, HTTPS incluído, CDN global, custo zero.

Domínio próprio: airtech.pt

Para usar um domínio personalizado em vez do URL do GitHub, o cliente precisa de registar airtech.pt num registador (ex: Namecheap, GoDaddy ou PTnic para .pt) e apontar o DNS para o GitHub Pages. O processo demora cerca de 24–48h. Podemos configurar isto em conjunto — é o passo que transforma o site de protótipo em presença digital profissional.

Google Analytics — opcional mas recomendado

Adicionar o tag do Google Analytics 4 permite saber quantas pessoas visitam o site, de onde vêm, que secções lêem e quando saem. Com esse dado, é possível optimizar o site ao longo do tempo com base em comportamento real dos clientes — não em suposições.
